# Mark Zuckerberg Returns to Writing Code After 20 Years, Powered by Claude Code

In a striking symbol of AI's ascendancy in software development, Mark Zuckerberg—the Meta CEO who hasn't written production code in two decades—is shipping code changes again. And he's doing it with Claude Code.

According to The Pragmatic Engineer, Zuckerberg has begun landing diffs—shipping code commits—after a two-decade hiatus. The move validates AI-assisted development workflows at the executive level, testing them before mandating company-wide adoption.

Zuckerberg predicted: "Within 12 to 18 months, most of the code is written by AI. This won't be simple autocomplete, but rather AI agents that can set goals, run tests, find problems, and write better code than top engineers."

## Meta's AI-Native Push

Meta has set aggressive targets: 50 to 80 percent AI-assisted code writing by mid-2026, with 65 percent of engineers required to write more than 75 percent of their committed code using AI assistance. These are tied to compensation and advancement metrics.

Claude Code, which Zuckerberg uses, has achieved over $2.5 billion in annualized run-rate revenue as of February 2026, reflecting massive enterprise adoption.

## Implications for Developers

If Meta achieves its targets, a significant portion of mid-level and junior engineering work could be automated. The engineers who thrive will be those who can effectively collaborate with AI agents, directing them toward business goals and handling exceptions.

## What to Watch

Zuckerberg's personal involvement means the executive suite has direct insight into which engineering roles are most vulnerable to automation—and which remain indispensable.
